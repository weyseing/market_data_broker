The pitch

A real-time data hub for crypto markets.

One Coinbase WSS connection upstream, many heterogeneous consumers downstream — plain WebSocket clients, MCP-speaking LLM agents, internal ops endpoints. Demand-driven via ref-counting in the registry: the broker only holds a Coinbase subscription while at least one consumer wants it. The moment the last consumer leaves, we send UNSUBSCRIBE upstream within milliseconds. Single-process Python asyncio. In-memory state only — restart-safe by design.

What it does

The broker makes your market-data setup simpler.

Fan out

One Coinbase WebSocket in, N heterogeneous consumers out — plain WS, MCP tools, ops endpoints. Bounded asyncio.Queue per consumer (default 1000). On overflow we drop the oldest message and increment a per-consumer counter. Sustained drops > 100 close that consumer with WS close code 1013 — slow consumers cannot degrade peers.

Demand-driven by design

We only ask Coinbase for a topic when at least one consumer wants it. The registry fires on_first_subscriber at the 0→1 edge and on_last_unsubscriber at the 1→0 edge — ingest hooks both into upstream SUBSCRIBE / UNSUBSCRIBE frames. Pinned by test_disconnect_releases_topics_no_leak.

Two surfaces, one core

Plain WebSocket for general clients on :8765. HTTP MCP for LLM agents on :8000 — five tools: list_topics, describe_topic, get_snapshot, stream_topic, get_hub_status. Both surfaces are first-class consumers — same topic taxonomy, same registry, same refcount semantics.

How it works

Four layers, one hinge.

Each layer owns one concern. ingest talks to Coinbase via the websockets library — exp backoff, stale-feed watchdog, demand reconcile. bus is asyncio pub/sub with bounded queues. registry ref-counts holders and fires sub/unsub callbacks. Surfaces speak WS or HTTP MCP via FastMCP. Adding a venue is one new ingest module; adding a protocol is one new server module. The other three layers don't change.

Topic taxonomy

Subscribe to anything in three dots.

Topics are dotted strings — {venue}.{channel}.{product_id}. Venue-first leaves room for binance.* without schema changes. Payloads are JSON Message envelopes; all numerics are strings end-to-end (Decimal precision — no float loss). Tools tell LLMs to use Decimal before any arithmetic.

The hinge

No leaked upstream subscriptions.

The registry holds dict[topic, set[consumer_id]]. The first holder triggers on_first_subscriber(topic) — ingest sends a Coinbase SUBSCRIBE frame. The last holder release triggers on_last_unsubscriber(topic) — ingest sends UNSUBSCRIBE. Both callbacks are async and run on the same event loop, no locks. The snapshot cache only watches the bus directly, not the registry — so it caches state without creating artificial demand.

t = 0
upstream subs: [ ]
t = 1 — client A subscribes BTC ticker
upstream subs: [ ticker.BTC-USD ]
t = 2 — clients B, C join
upstream subs: [ ticker.BTC-USD ] // still 1 upstream Coinbase sub · ref-count = 3
t = 3 — all leave
upstream subs: [ ] // last holder gone → on_last_unsubscriber → UNSUBSCRIBE upstream

Three clients, two topics, zero leaks.

Trade-offs

What's deliberate, what's deferred.

In-memory only · no database, no persistence A streaming pipe, not a database. On restart we lose the snapshot cache + consumer subs; ingest reconciles upstream from new demand. Documented in the README. +
Drop-oldest backpressure, per consumer A slow consumer cannot stall the broker or peers — its own backlog gets trimmed. Drops surface per-consumer in /status. Sustained drops > 100 close the WS with code 1013. +
Single venue today — Coinbase only Topic strings are venue-prefixed. Adding binance.* is one new ingest module that publishes the same envelope — bus, registry, snapshot, and surfaces don't change. +
No auth on public surfaces Designed for an internal trusted network. JWT or mTLS would live at the server-surface layer — bus and registry stay protocol-agnostic. +
Vertical scale only Horizontal scale would shard by topic behind a broker like NATS or Redis Streams. Out of scope for the MVP; the layered design leaves a clean seam for it. +
